# Colors

The Boulder UI color palette is designed to provide a consistent and accessible visual experience. Colors are defined as CSS custom properties so that consumers can override any value to match their brand without touching component source code.

All color tokens are prefixed with `--boulder-color-`.

---

## Base Palette

The base palette is organized into two scales: **Olive** and **Sand**. These are the raw, foundational colors from which all semantic roles are derived. They are intentionally earthy and warm, reflecting the Boulder identity. Semantic tokens always reference a base scale variable — never a raw hex value — which means a single override at the scale level propagates correctly through the entire system.

## Customization

Override any token once in your application's `:root` to retheme the entire library:

```css
:root {
  /* Swap the primary brand color */
  --boulder-color-primary: #0070f3;
  --boulder-color-primary-hover: #005cc5;

  /* Adjust text contrast */
  --boulder-color-text: #111111;
  --boulder-color-text-secondary: #555555;
}
```

No component-level changes are needed. All components that reference these tokens will reflect the new values automatically.
