# Colors

The Boulder UI color palette is designed to provide a consistent and accessible visual experience across all applications. Our colors are defined as CSS custom properties (design tokens) to ensure easy customization and maintainability.

All color tokens are prefixed with `--boulder-color-`.

## Base Palette

These are the foundational colors from which the semantic palette is derived. They offer a neutral base for building more specific color roles.

## Usage

To use these colors in your CSS, reference them as CSS custom properties:

```css
.my-component {
  background-color: var(--boulder-color-primary);
  color: var(--boulder-color-text-inverse);
  border: 1px solid var(--boulder-color-border);
}
```

## Customization

You can easily override any of these color tokens in your application's CSS. For example, to change the primary color:

```css
:root {
  --boulder-color-primary: #007bff; /* A new blue primary color */
}
```

This allows for flexible theming and ensures that your application's visual identity can be adapted without modifying the library's source code.
